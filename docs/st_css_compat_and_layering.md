# The World × SillyTavern CSS 兼容与层级开发文档

> 版本：2026-03-27
> 目的：沉淀本次移动端动态天色适配经验，为后续“光效（阳光/月光光栅）”开发提供稳定的层级基线。

---

## 1. 本次问题与根因复盘

### 1.1 现象
- 移动端开启全局动态背景后：
  - 一度出现“白底看不到天色”；
  - 修复后又出现“天色压住天气特效”。

### 1.2 根因
1. **容器高度塌陷**：
   - `.tw-global-theme-container` 在某些移动端布局下出现 `rect.h = 0`，导致背景层虽存在但不可见。
2. **层级冲突**：
   - 天色层 z-index 提得过高后，盖住了天气特效层。
3. **body 背景回落策略不当**：
   - 非沉浸模式若 `body` 回到默认背景，可能覆盖视觉上的天色结果。

## 2. SillyTavern 核心层级基线（开发必须遵守）

以下数据来自本地 SillyTavern 样式（`/Users/macbookair/SillyTavern/public/...`）：

### 2.1 酒馆核心层
- `#bg1`：`z-index: -1`（背景图层）
- `#sheld`：`z-index: 30`（聊天壳层）
- `#chat`：`z-index: 30`（聊天主容器）
- `#top-bar`：`z-index: 3005`（顶栏）

### 2.2 The World 现有层
- `#the_world-fx-layer`：`z-index: 10000`（前景天气/粒子特效）
- `#the_world-fx-layer-bg`：`z-index: -1`（背景特效层）
- `.tw-global-theme-container`：当前已固定为 `z-index: -2`（全局天色层，最底）

## 3. 当前最终兼容策略（已落地）

### 3.1 天色容器固定全屏，避免移动端 100% 高度塌陷
在 `modules/core/GlobalThemeManager.js` 中：
- 容器与背景层均使用 `position: fixed` + 视口尺寸（`100vw/100vh`）；
- 容器采用 `top/left/right/bottom: 0`；
- 避免依赖父元素高度链。

### 3.2 层级归位
- 天色层：`z-index: -2`（在 `#bg1` 之下也可工作，且绝不压住天气层）；
- 天气层：保持 `#the_world-fx-layer` 的高层级（10000）。

### 3.3 移动端性能降级
- 关闭天色平移动画（仅保留渐变与淡变）；
- 缩短淡变时长；
- 沉浸模式下关闭移动端消息气泡 `backdrop-filter`。

---

## 4. 面向“光效（月光/阳光光栅）”的层级设计建议

> 目标：既能照亮背景/聊天区，又不破坏消息可读性与交互。

### 4.1 推荐分层
1. **全局背景光栅层（Global Light BG）**
   - 位置：在天色之上、聊天壳之下。
   - 建议 z-index：`1~20`（低于 `#sheld:30`）。
   - 用途：大范围扫光、天光方向、云隙光。

2. **聊天内局部光效层（Chat Light Overlay）**
   - 位置：绑定 `#chat` 内部（或消息列表容器上方）。
   - 用途：仅影响消息区域氛围，不影响输入栏和顶栏。

3. **天气特效层（保持现状）**
   - 继续在高层，保证雨雪、花瓣、雷光不被盖住。

### 4.2 关键 CSS 约束
- 所有光效层默认 `pointer-events: none`；
- 优先使用 `opacity + transform` 做动画（避免频繁触发重排）；
- `mix-blend-mode` 仅作增强，必须提供 fallback（部分移动浏览器兼容不稳定）；
- 光效不应直接改 `#chat` 的 `z-index`，以免影响酒馆内部交互逻辑。

---

## 5. 与 `#chat` / message 容器联动时的实现建议

### 5.1 可操作目标
- 聊天容器：`#chat`
- 消息主体：`.mes_content`

### 5.2 推荐做法
1. **先做容器级光效**（`#chat` 上方半透明渐层/扫光）
2. **再做消息级微光**（仅对 `.mes_content` 添加很弱的 `box-shadow` 或渐层）
3. 保持文字对比度兜底：
   - 深背景时强制亮字色；
   - 光效高亮时降低光栅 alpha，避免“发灰发白”导致阅读困难。

### 5.3 避免事项
- 不要对 `.mes_content` 长时间使用高强度 `backdrop-filter`（移动端掉帧明显）；
- 不要把全局光效层放到 `z-index >= 30`（会与 `#chat/#sheld` 争层）。

---

## 6. 调试 SOP（后续遇到层级问题直接走）

1. 看容器几何：是否 `rect.h > 0`。
2. 看层级：`z-index` 是否符合“天色 < 聊天 < 天气特效”。
3. 看可见性：`display/opacity/visibility` 是否被覆盖。
4. 强制注入测试渐变，确认是“渲染失败”还是“被遮挡”。
5. 临时关闭天气层验证遮挡方向。

- `css/base.css`
  - `#the_world-fx-layer` / `#the_world-fx-layer-bg`：天气特效层定义

1. 先实现 **Global Light BG**（不碰消息结构）。
2. 再实现 **Chat Light Overlay**（仅挂 `#chat` 内部）。

---

## 9. 结论

本次兼容验证后，稳定层级原则可归纳为：

- **天色永远最底（-2）**
- **酒馆聊天结构保持原层级（30 / 3005）**
- **天气特效保持高层（10000）**
- **光效作为中间层按功能拆分，不跨层乱改 z-index**

按这个原则推进“月光/阳光光栅”，可以最大化降低与酒馆原生 CSS 的冲突风险。
